Y Combinator

The first thing that mother Google shows on a search for Y Combinator is the startup seed accelerator. There is significantly less (and lower priority) results about what the real meaning behind the Y Combinator is.

One of the best resources on the Y Combinator (no surprises there) describes the Y Combinator as a particular fixed-point combinators that was discovered by our beloved Haskell Curry. The definition for a fixed-point combinator, g, is g(f) = f(g(f)). What's super cool is that you can easily keep applying that equation to itself, yielding g(f) = f(g(f)) = f(f(g(f))) = f(f(f(...)))! To infinity and beyond! In practice, a fixed-point combinator can be used to support anonymous recursive functions, allowing recursion to happen without binding functions to identifiers.

The Y Combinator is defined as Y = λf.(λx.f (x x)) (λx.f (x x)). The Javascript implementation of the Y Combinator on this page demonstrates how the factorial function can be defined without referencing it ever having to reference itself:

  var yCombinator = function(f) {
    return (function(x) {
      return f(function() {
        return (x(x)).apply(null, [].splice.call(arguments, 0));
      });
    })(function(x) {
      return f(function() {
        return (x(x)).apply(null, [].splice.call(arguments, 0));
      });
    });
  };

  var factorial = yCombinator(function(fac) {
    return function(n) {
      if (n === 0) {
        return 1;
      } else {
        return n * fac(n-1);
      }
    }
  });

  /* 
  // Recursive factorial function that references itself
  var factorial = function(n) {
    if (n === 0) {
      return 1;
    } else {
      return n * factorial(n-1); // Bingo!
    }
  };
  */

  console.log(factorial(5));  // 120
  console.log(factorial(6));  // 720
  console.log(factorial(10)); // 3628800

The above implementation is in fact a slight variation of the Y Combinator called the Z Combinator, where Z = λf.(λx.f (λv.((x x) v))) (λx.f (λv.((x x) v))). The Z Combinator is just like the Y Combinator except an additional function is added ("Eta-Expansion") to stop the Y Combinator from evaluating until the resulting function is called.
